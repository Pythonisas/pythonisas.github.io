* Architecture Summary

** Tech Stack
- Generator: Hugo (v0.128.0, static site generator)
- Source Format: Org-mode (blog.org) → Markdown (.md) via ox-hugo
- Theme: smol (minimal, privacy-first, no-JS, monospace, dark mode)
- Hosting: GitHub Pages
- Deployment: GitHub Actions (automated on push to main)
- Versioning: Git

** Workflow
1. Write content in *blog.org* using Org-mode (single master file)
2. Export sections to Markdown via ox-hugo property drawers
3. Hugo generates static site from markdown + smol theme
4. GitHub Actions builds and deploys to GitHub Pages

* The Master File: content/blog.org

** Key Facts
- 3,742 lines — ALL content in one Org file
- Hugo export headers: =#+hugo_base_dir: ../= and =#+hugo_section: [pages|posts]=
- Each section uses PROPERTIES drawers:
  - =:EXPORT_FILE_NAME:= → maps heading to .md filename
  - =:EXPORT_HUGO_PUBLISHDATE:= → publication date
  - =:EXPORT_HUGO_SECTION:= → target section (posts or pages)

* Literate Programming Potential

** Current State
- 62 src blocks in blog.org — all educational, zero =:tangle= headers
- No tangled companion scripts
- Code blocks serve as documentation, not executable artifacts

** Opportunities
- Tangle bash/conf blocks → runnable scripts for readers
- Tangle yara blocks → detection rule files
- Tangle ansible blocks → playbooks
- Single source of truth: blog post = documentation + code
- Compare with Pythonisas approach (org-static-blog + tangle)

* Key Takeaways for Pythonisas / ValdeLAB36

1. *Single master .org file* pattern — Eduardo writes everything in one blog.org (3,742 lines). Pros: single source, easy search. Cons: massive file, merge conflicts.
2. *ox-hugo vs org-static-blog* — Eduardo uses Hugo as the renderer, Org as the authoring tool. We use Org as both author and renderer.
3. *Privacy-first theme* — smol has zero JS/tracking. Our Pythonisas uses highlight.js CDN (small trade-off).
4. *LLM experiments* — Ollama + Podman + MCP agents pattern directly applicable to ValdeLAB36.
5. *IaC with Emacs* — Ansible playbooks authored in Org-mode, a pattern we could adopt.
6. *No tangling yet* — Eduardo hasn't tapped Org Babel's full literate programming power. We're ahead with our faker post's =#+begin_src python= blocks.
